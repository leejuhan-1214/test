# ISHS ARENA v3

직접 등록한 몬스터로 CPU 또는 친구와 대전하는 한국어 웹 배틀 게임입니다. v3부터 계정, 서버 공용 도감, 계정별 팀 저장, 6자리 프라이빗 룸과 서버 판정 멀티플레이가 추가되었습니다.

## 가장 쉬운 실행 방법

1. [Node.js 20 이상](https://nodejs.org/)을 설치합니다.
2. ZIP의 압축을 푼 뒤, **WebStorm이나 브라우저가 아닌 Windows 파일 탐색기**에서 `START-ISHS-ARENA.cmd`를 더블 클릭합니다.
3. 브라우저에서 `http://localhost:4173`을 엽니다.
4. 처음 접속하면 **계정 만들기**에서 닉네임과 8자 이상의 비밀번호를 등록합니다.

`index.html`을 직접 더블 클릭하면 서버가 없으므로 계정·공용 도감·멀티플레이는 작동하지 않습니다. 이때는 안내 화면의 **오프라인 CPU 모드 계속하기**만 사용할 수 있습니다.

터미널에서 실행하려면 다음 명령도 사용할 수 있습니다.

```text
npm start
```

## 친구와 같은 Wi-Fi에서 플레이

1. 서버를 실행한 컴퓨터에서 `ipconfig`로 IPv4 주소를 확인합니다.
2. 친구가 `http://서버컴퓨터IP:4173`으로 접속합니다.
3. 각자 계정을 만들고 팀을 편성합니다.
4. 한 명이 **프라이빗 룸**을 만들고 6자리 코드를 전달합니다.
5. 다른 사람이 코드를 입력한 뒤 양쪽 모두 준비를 누르면 배틀이 시작됩니다.

Windows 방화벽이 연결을 막는 경우 Node.js의 개인 네트워크 통신을 허용해야 합니다.

## 계정과 공용 도감

- 닉네임은 한글·영문·숫자·밑줄 2~16자입니다.
- 비밀번호는 8자 이상이며 서버에는 원문 대신 무작위 salt가 포함된 scrypt 해시로 저장됩니다.
- 로그인 쿠키는 HttpOnly·SameSite 방식이며 7일 동안 유지됩니다.
- 몬스터 도감은 서버에 한 벌만 저장되어 모든 계정에서 같은 내용이 보입니다.
- 도감 등록·수정·삭제는 관리자 비밀번호 인증 후에만 서버가 허용합니다.
- 팀 편성, 특성, 기술과 노력치는 계정별로 따로 저장됩니다.
- 계정·도감·팀 데이터는 게임 코드와 분리된 `C:\Users\사용자이름\.ishs-arena-data\store.json`에 저장됩니다.
- 저장할 때마다 직전 정상 데이터가 `store.backup.json`으로 자동 백업됩니다.
- 예전 버전의 `server-data/store.json`이 있으면 첫 실행 때 새 영구 저장 위치로 자동 이전합니다.

## 배틀 기능

- CPU 싱글·더블 배틀
- 프라이빗 룸 1 대 1 싱글 배틀
- 기술별 현재 PP / 최대 PP 표시 및 실제 소모
- 모든 PP가 소진되면 `발버둥` 사용
- 기술 버튼을 18개 타입 색으로 구분
- 기술 타입, 물리·특수·변화 분류, 위력, 명중률과 한국어 효과 설명 표시
- 타입 상성에 따라 `효과가 굉장했다!`, `효과가 별로인 듯하다…`, `효과가 없다…` 표시
- 급소, 명중 실패, 상태이상, 능력치 변화, 회복, 흡수와 반동 메시지
- **상태** 버튼: 공격·방어·특공·특방·스피드 랭크와 적용된 현재 능력치 확인
- **포켓몬** 버튼: 기절하지 않은 벤치 몬스터로 교체
- 교체 우선 처리, 등장 특성, 상태이상 턴 데미지와 기절 후 강제 교체
- 화면 전체를 다시 밝게 번쩍이던 피격 효과 제거, HP 변화와 작은 이동 효과만 사용
- 서버가 양쪽 행동을 받은 뒤 순서와 데미지를 판정하며 상대의 선택 내용은 공개하지 않음

## 기술·특성 데이터

- 온라인에서는 Pokémon Showdown의 기술·특성 원본 데이터와 한국 포켓몬 위키의 번호별 한국어 이름을 합쳐 불러옵니다.
- 같은 번호 또는 같은 영문 식별자를 가진 항목은 하나로 합쳐 중복을 줄였습니다.
- 기술 설명은 구조화된 효과를 바탕으로 한국어 문장으로 표시합니다.
- 처음 받은 전체 데이터는 브라우저에 캐시됩니다.
- 인터넷 연결이 없으면 정밀 구현된 내장 기술 51개와 특성 24개를 사용합니다.
- 일반 데미지, 랭크 변화, 상태이상, 선제공격, 회복, 흡수, 반동, 연속공격과 여러 공통 특성은 엔진에 반영됩니다.
- 날씨·필드·아이템·폼 변화·특수 전용 연출처럼 별도 시스템이 필요한 일부 고유 효과는 데이터 설명은 표시되지만 완전한 공식 판정과 다를 수 있습니다.

## 도감 및 팀 편성

- 도감 등록 시 이미지, 이름, 타입, 레벨, 종족값, 개체값(IV)을 저장합니다.
- 해당 몬스터가 사용할 수 있는 기술과 특성을 도감에서 먼저 허용합니다.
- 팀에 불러올 때 허용된 특성 하나와 기술 최대 4개를 고릅니다.
- 노력치(EV)는 능력치별 0~252, 합계 510 이하로 설정합니다.
- 종족값·개체값·노력치·레벨을 능력치 공식에 반영합니다.
- 한 계정의 팀은 최대 6마리입니다.

## 인터넷에 사이트로 배포

이 버전은 서버 기능이 있으므로 GitHub Pages 같은 정적 호스팅만으로는 멀티플레이할 수 없습니다. Node.js 서버와 영구 저장소를 제공하는 서비스가 필요합니다. 자세한 과정은 `ONLINE-DEPLOY.md`에 정리되어 있습니다.

- 시작 명령: `npm start`
- 필요한 Node.js 버전: 20 이상
- 서비스 포트: 호스팅 서비스가 제공하는 `PORT` 환경 변수를 자동 사용
- Render 배포 설정: 포함된 `render.yaml`이 무료 웹 서비스를 생성합니다.
- 데이터 보존: 온라인에서는 무료 Neon Postgres에 계정·도감·팀을 저장합니다. Render의 `DATABASE_URL`에 Neon 연결 문자열을 설정합니다.
- 로컬 보존: 게임 폴더 밖의 `.ishs-arena-data`를 사용하므로 새 ZIP으로 코드 폴더를 교체해도 유지됩니다.
- 관리자 비밀번호: 로컬 실행 시 시작 창에서 입력하고, Render 배포 시 `ISHS_ADMIN_PASSWORD` 비밀 환경값으로 설정합니다. 비밀번호는 소스 코드나 GitHub에 저장되지 않습니다.
- 공개 운영 시에는 Render가 제공하는 HTTPS 주소를 사용하며, 데이터베이스 연결 문자열은 GitHub에 올리지 않습니다.

## 파일 구성

- `index.html` — 화면과 메뉴
- `styles.css` — 전체 디자인, 전투 UI, 모바일 화면
- `app-v2.js` — 도감, 팀 편성, CPU 배틀, 기술·특성 데이터
- `multiplayer.js` — 계정 화면, 공용 데이터 동기화, 프라이빗 룸 UI
- `server.js` — 계정·공용 도감·팀·룸 API와 정적 사이트 서버
- `battle-engine.js` — 서버 권한 멀티플레이 전투 판정
- `START-ISHS-ARENA.cmd` — Windows 간편 실행
- `render.yaml` — 외부 웹사이트와 영구 디스크 배포 설정
- `ONLINE-DEPLOY.md` — 각 컴퓨터에서 접속하는 사이트 배포 설명서
- `package.json` — Node.js 실행 설정

## 참고

이 프로젝트는 개인 창작·학습용 오리지널 프로토타입입니다. 공식 Pokémon 로고, 캐릭터 이미지, 음악과 화면 자산은 포함하지 않았습니다. 공식 게임의 조작 흐름과 배틀 정보 구성을 참고했지만 상표·저작권 문제가 있는 화면을 그대로 복제하지 않았으며, 모든 세대의 예외 규칙을 1:1로 재현한 공식 게임은 아닙니다. 몬스터 이미지에는 직접 만들었거나 사용 권한이 있는 자료를 사용하세요.
